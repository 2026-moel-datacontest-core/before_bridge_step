## 문서 목적

- 강사님께 프로젝트 아이디어와 현재 구현 수준을 빠르게 검토받기 위한 요약 문서

## 1. 프로젝트 한 줄 소개

`law-main-road`는 계약 단계 위험 탐지와 분쟁 발생 후 대응 가이드를 하나의 흐름으로 연결하는 근거 기반 노동권 보호 AI입니다.

## 2. 해결하려는 문제

- 많은 사용자가 근로계약, 해고, 임금 체불, 육아휴직, 가족돌봄휴가 같은 노동 문제를 겪어도 관련 법과 절차를 바로 이해하기 어렵습니다.
- 이 문제는 일반 근로자에게도 발생하지만, 외국인 근로자, 장애인 근로자, 사회초년생, 비정형 노동자처럼 정보 접근성이 낮은 사용자에게 더 크게 나타납니다.
- 사용자는 법 조문 원문 자체보다, "지금 내 상황에서 어떤 권리가 있고 무엇을 먼저 해야 하는지"를 빠르게 알고 싶어 합니다.
- 일반적인 생성형 AI는 실제 검색 결과에 없는 조문을 섞어 말할 위험이 있어, 노동법 안내 도구로 사용할 때 신뢰 문제가 큽니다.

## 3. 제안하는 해결 방식

### Before

- 근로계약서 또는 OCR 결과를 입력받아 주요 항목을 읽고
- 누락, 불명확, 주의 필요, 위법 가능성 신호를 분류하며
- 관련 조문을 함께 제시합니다.

### Bridge

- 계약 단계에서 발견된 위험 태그와 요약을 저장해 두고
- 이후 분쟁이 발생했을 때 "계약 단계에서 이미 예견 가능했던 위험"으로 연결합니다.

### After

- 사용자가 사고, 해고, 체불, 휴직 거절 같은 상황을 자유롭게 서술하면
- 사건을 구조화해서 관련 조문을 검색하고
- 근거가 있는 다음 행동과 증거 수집 방향을 안내합니다.

### Recovery

- 회복 및 후속 문서 작성 지원 단계입니다.
- 현재 MVP 핵심 범위는 아니며, 발표에서는 확장 가능성으로 설명합니다.

## 4. 이 프로젝트의 차별점

- 범용성 + 취약계층 확장성: 일반 사용자도 사용할 수 있지만, 외국인 근로자, 장애인 근로자, 돌봄 상황 사용자처럼 정보 취약성이 큰 경우까지 포괄할 수 있습니다.
- 계약 단계와 분쟁 단계를 연결: `Before -> Bridge -> After` 흐름으로 "예방"과 "대응"을 한 제품 안에서 설명할 수 있습니다.
- 시나리오 범위가 넓음: 현재 시나리오도 외국인 계약 문제, 최저임금/수습, 장애인 편의제공, 부당해고/체불, 육아휴직/가족돌봄휴가까지 아우르고 있습니다.
- 검색 결과와 grounded context에 없는 조문을 명시적으로 인용하면 검증 단계에서 실패 처리되도록 설계해 환각을 줄였습니다.
- 실제 시나리오 검증 기반: 데모 문안을 먼저 만들고, 각 시나리오가 현재 corpus로 가능한지 검증해 범위를 조정했습니다.

## 5. 현재 구현 수준

### 5-1. 공통 기반

| 영역 | 현재 상태 |
|---|---|
| 데이터 | `backend/data/law_chunks/all_chunks.json` 기준 `1722` chunks 고정 |
| DB | PostgreSQL `law_chunks` `1722`건, embedding `1722 / 1722` 완료 |
| Retrieval | `POST /api/v1/retrieve` 구현 완료, `hit@5 = 60/60 (100.00%)` |
| 검증 기반 | eval runner, scenario smoke, citation 검증 구조 확보 |

이 공통 기반 위에서 `Before`와 `After` 기능을 각각 붙이는 구조로 개발 중입니다.

### 5-2. After 단계 구현 수준

| 영역 | 현재 상태 |
|---|---|
| Answer | `POST /api/v1/answer` 구현 완료, `citation_grounding_clean = 60/60` |
| 응답 구조 | 검색 결과에 없는 조문 인용 금지, `cited_articles` 포함 응답 고정 |
| 평가 | baseline answer eval `137/153`, partial/failure `16` |
| 시나리오 검증 | `SCN-001`, `SCN-004`, `SCN-005` After 핵심 데모 안정화 |
| 문서 초안 | `POST /api/v1/documents/draft` 구현 완료, SCN-004 진정서/이유서 초안 지원 |
| Frontend | Next.js `/after` 4-route demo flow 구현 완료, copy/print 포함 |

현재 저장소 기준으로는 `After` 단계의 핵심 backend 로직과 검증 자산이 먼저 구현된 상태입니다.

### 5-3. Before 단계 구현 수준

| 영역 | 현재 상태 |
| :--- | :--- |
| 파이프라인 아키텍처 | Phase A(표준 항목 매핑) 및 Phase B(OCR→비교→검증→LLM) 구조 설계 및 코드 초안 작성 완료 |
| OCR 및 구조화 | API 연동을 통한 계약서 이미지 2-Layer OCR 테스트 완료 및 JSON 구조화 포맷(`structured`, `raw_sections`) 치환 확인 |
| 법령 검색(Retrieval) | OCR로 추출·구조화된 JSON 데이터를 기반으로 관련 법 조항 및 청크(`law_chunks`) 추출 테스트 완료 |
| 단위 로직 구현 | LLM 환각 통제를 위한 수치 검증 로직(`rule_validator.py`) 및 표준 항목 대조 로직(`section_comparator.py`) 초안 확보 |
| 테스트 파이프라인 | T-0(상수/출력 검증)부터 T-7(SCN-002 시나리오 검증)까지 단계별 단위/통합 테스트 환경 구축 완료 |

현재 구현된 코드 초안과 API 테스트 결과를 바탕으로, 구축해둔 T-0 ~ T-7 테스트 파이프라인을 따라 본격적인 모듈별 단위 테스트 및 전체 통합 테스트에 돌입할 예정입니다.


## 6. 대표 데모 시나리오

| 시나리오 | 유형 | 현재 상태 | 발표 활용도 |
|---|---|---|---|
| `SCN-001` 외국인 계약서/기숙사/차별/사업장 변경 | `Full` | 커버됨, `top_k=10` 데모 경로에서 안정화 | 제품 스토리 대표 후보 |
| `SCN-004` 카톡 해고 및 임금·퇴직금 체불 | `After` | frontend demo 구현 완료 | 실제 frontend main demo |
| `SCN-005` 육아휴직 및 가족돌봄휴가 거절 | `After` | 커버됨 | 사회적 공감도가 높은 보조 시연 후보 |
| `SCN-003` 장애인 편의제공 및 지원 제도 안내 | `Before` | 최소 데이터 보강 후 커버됨 | 확장성 설명용 후보 |
| `SCN-002` 최저임금/수습 꼼수 | `Before` | `Partial` | 설명형 데모까지만 권장 |

제품 스토리용 메인 후보는 `SCN-001`이고, 현재 실제 frontend main demo는 `SCN-004`가 적절합니다. `SCN-004`는 `/after`에서 권리 안내와 문서 초안 생성까지 end-to-end로 시연 가능하며, preset 경로는 `top_k=10`, `ef_search=100`을 명시합니다. `SCN-001 Full`은 API smoke 또는 제품 스토리 설명용으로 유지합니다.

## 7. 기대 효과

- 일반 사용자도 계약 전 위험과 사후 대응 방향을 더 빠르게 이해할 수 있습니다.
- 정보 취약성이 큰 사용자에게는 더 직접적인 보호 효과를 줄 수 있습니다.
- 분쟁 발생 후에도 법 조문 근거와 함께 대응 순서를 정리받을 수 있습니다.
- 공공기관, 상담 보조 인력, 지원 활동가에게도 보조 도구로 확장 가능합니다.
- 단순 Q&A가 아니라 "근거 제시 + 다음 행동 안내" 구조를 보여줄 수 있습니다.

## 8. 강사님께 확인받고 싶은 포인트

- 제품 스토리는 `SCN-001`, 실제 frontend demo는 `SCN-004`로 두는 구성이 설득력 있는지
