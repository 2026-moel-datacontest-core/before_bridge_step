# Pipeline Secret Dependency Audit

기준일: `2026-04-21`

공통 전제:

- 모든 수정은 현재 사용자에게 보이는 UI 구조와 기존 기능 동작을 유지하는 범위에서만 진행한다.
- 정리, 보안 보강, 안정화, 리팩터링이 필요하더라도 현재 확인 가능한 화면 흐름과 기능 결과를 바꾸지 않는 것을 우선한다.

목적:

- 현재 main 안에 합쳐진 `before` / `after` 파이프라인의 실제 런타임 의존성을 분리해서 본다.
- secret, env, 외부 API, 로컬 파일 의존성 중 무엇이 실제로 필요한지 정리한다.
- 지금 바로 실행을 막는 blocker 와, 추후 실제 API 연결 시 필요한 값을 구분한다.

---

## 1. 요약 결론

현재 상태를 한 줄로 정리하면:

- `before`는 **지금 main 안에서는 mock-only UI** 이므로 Vertex AI secret 이 필요하지 않다.
- `after`는 **live 경로에서 backend + DB + Vertex AI** 에 의존한다.
- 다만 현재는 Vertex AI secret 이전에 **`backend/.env` 부재로 `DATABASE_URL`이 없어 backend import 자체가 실패**한다.

즉, 지금 당장 실행 blocker 는 `after backend` 쪽이고, `before`는 실제 API 연결을 아직 하지 않았기 때문에 secret blocker 가 아니다.

- `frontend/src/app/before/page.tsx`
- `frontend/src/lib/before-api.ts`

- `before_web` 원본 backend 를 합치지 않았기 때문에 Vertex AI 인증이 실제로는 사용되지 않는다.

현재 `/before` 관련 blocker:

- 없음

현재 `/before` 관련 주의사항:

- UI는 `before_web`를 닮았지만 backend 는 연결되어 있지 않다.
- 지금 `/before`는 “실제 계약서 분석 서비스”가 아니라 “mock 흐름 데모”다.

## 5. after 실행 경로별 실제 필요 조건

### A. `/after` exact preset path

관련 파일:

- `frontend/src/app/after/page.tsx`
- `frontend/src/lib/scenarioPresets.ts`

특징:

- fixed answer preset 을 사용하면 `/api/v1/answer`를 호출하지 않는다.

필요 조건:

- 프론트만 있으면 됨
- backend 불필요
- Vertex AI 불필요
- DB 불필요

### B. `/after` live answer path

관련 파일:

- `frontend/src/lib/api.ts`
- `backend/app/routers/answer.py`
- `backend/app/services/answer_generation.py`
- `backend/app/services/retrieval.py`
- `backend/app/services/embedding.py`

필요 조건:

- backend server 기동 가능
- `DATABASE_URL`
- PostgreSQL + pgvector
- `GCP_LOCATION`
- `GCP_PROJECT` 또는 ADC project
- `GOOGLE_APPLICATION_CREDENTIALS` 또는 ADC

즉:

- DB 와 Vertex AI 둘 다 필요

### C. `/after` document draft path

관련 파일:

- `frontend/src/lib/api.ts`
- `backend/app/routers/document_draft.py`
- `backend/app/services/document_draft.py`

논리적 필요 조건만 보면:

- `document_draft.py` 자체는 Vertex AI 를 직접 호출하지 않는다.
- DB 도 직접 호출하지 않는다.

하지만 실제 현재 구조에서는:

- `backend/main.py` 가 `api_router` 전체를 include 한다.
- `backend/app/routers/__init__.py` 가 `answer_router`, `retrieval_router`, `document_draft_router`를 한 번에 import 한다.
- `answer_router` import 과정에서 `backend.app.services` → `answer_generation` → `retrieval` → `backend.app.db` 로 이어진다.
- `backend/app/db.py` 는 import 시점에 `DATABASE_URL`이 없으면 바로 예외를 raise 한다.

즉 결론:

- 문서 초안 기능 자체는 secret 이 없어도 될 수 있지만,
- **현재 앱 구조상 backend 전체가 먼저 뜨지 못하기 때문에 실제로는 `DATABASE_URL` 없이 문서 초안 API도 접근 불가**

## 6. 실제 확인된 blocker

### blocker 1. `backend/.env` 없음

확인:

- `backend/.env` 현재 없음
- `backend/.env.example`만 존재

영향:

- `from backend.main import app` 단계에서 실패

실제 오류:

- `RuntimeError: DATABASE_URL is not set. Create backend/.env from backend/.env.example.`

### blocker 2. backend import 가 retrieval / answer 경로를 함께 끌어옴

영향:

- `document_draft`만 쓰고 싶어도 앱 시작 단계에서 DB 요구사항을 우회할 수 없음

### blocker 3. live answer 경로는 Vertex AI 인증 필요

관련:

- `backend/app/services/embedding.py`
- `backend/app/services/answer_generation.py`

영향:

- DB 문제를 해결한 뒤에도
  - `GCP_LOCATION`
  - `GCP_PROJECT` 또는 ADC project
  - `GOOGLE_APPLICATION_CREDENTIALS` 또는 ADC
  가 없으면 `/api/v1/answer`는 실패

## 8. 문제 리스트

### before

1. 현재 main 안의 `/before`는 mock-only 이므로 secret 문제는 없음
2. 원본 before_web 에는 Vertex AI secret 구조가 있었지만 실제 secret 파일은 현재 확인되지 않음
3. 나중에 before 실제 API 를 다시 붙일 경우
   - `GCP_PROJECT_ID`
   - `GCP_LOCATION`
   - `GOOGLE_APPLICATION_CREDENTIALS`
   - 원본 assets 파일들
   이 필요

### after

1. `backend/.env` 부재로 `DATABASE_URL` 없음
2. 이 때문에 backend import 자체가 실패
3. 결과적으로 `/api/v1/answer` 뿐 아니라 `/api/v1/documents/draft`도 현재 구조상 접근 불가
4. DB 문제 해결 후에는 `/api/v1/answer` live path 에 Vertex AI 인증이 추가로 필요
5. 프론트 exact preset path 는 backend 없이도 동작하지만, live path 와 draft path 는 backend 기동이 전제

---

## 9. 우선 대응 순서

### 1순위

`backend/.env` 를 준비해서 backend import blocker 를 먼저 해소한다.

최소 필요:

- `DATABASE_URL`
- `GCP_LOCATION`
- `GCP_PROJECT` 또는 ADC project

### 2순위

`/api/v1/answer` live path 용 Vertex AI 인증 상태를 확인한다.

- `GOOGLE_APPLICATION_CREDENTIALS` 경로
또는
- ADC (`gcloud auth application-default login`)

### 3순위

필요하면 backend import 구조를 나중에 분리 검토한다.

목표:

- document draft 만 쓸 때 retrieval / answer import 때문에 전체 backend 가 막히지 않게 하는 구조 개선 가능성 검토

### 4순위

`before` 실제 API 연결 여부를 별도 결정한다.

- 지금은 mock 유지
- 실제 연결 시에만 원본 before_web 의 Vertex / asset / secret 체계를 다시 검토

---

## 10. 현재 판단

현재 가장 중요한 사실은 이거다.

- **지금 당장 문제는 before secret 이 아니라 after backend 기동 조건이다.**
- before 는 현재 mock-only 라서 secret 없이도 돌아간다.
- after 는 Vertex AI 이전에 `DATABASE_URL` 부재가 먼저 막고 있다.
